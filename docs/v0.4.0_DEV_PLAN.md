### 1.2 同义词匹配引擎

**产出文件：**
- `core/synonyms.py`（新文件）
- `data/synonyms.yaml`（新文件，同义词定义）
- `core/matcher.py`（修改：`_keyword_score` 扩展）

**设计原则：**
- **不降低精确匹配得分**：精确匹配仍为 +2.0，同义词匹配 +1.0，模糊匹配 +0.5
- 短语级同义词：`"减肥" → ["减脂", "瘦身", "减重"]`
- 编辑距离模糊匹配：Levenshtein 距离 ≤ 2 的中文词或 ≤ 3 的英文词 → 加分
- 按需加载，同义词字典量级控制在 ~200 条内

**`core/synonyms.py` API：**

```python
class SynonymEngine:
    def __init__(self, yaml_path: str = "data/synonyms.yaml"):
        """加载同义词库"""

    def expand(self, word: str, limit: int = 5) -> list[str]:
        """扩展单个词 → 同义词列表（含自身）"""

    def score(self, word: str, candidate: str) -> float:
        """
        返回匹配度分数：
        - 1.0 = 精确匹配
        - 0.8 = 同义词匹配
        - 0.4 = 编辑距离模糊匹配
        - 0.0 = 无匹配
        """

    def expand_keywords(self, keywords: list[str]) -> list[str]:
        """扩展关键词列表 → 去重后的扩大列表"""
```

### 2D. 自动节奏卡点

> **新增依赖：** `librosa>=0.10.0` + `soundfile>=0.12.0`  
> **依赖理由：** FFmpeg 无法提供精准的 BPM 和节拍时间戳。librosa 是 Python 音频分析标准库，已内置于多个类似视频编辑工具的依赖中。

**产出文件：** `core/rhythm.py`（新文件）+ `core/renderer.py`（修改）+ `core/pipeline.py`（修改）

**`core/rhythm.py` API：**

```python
class RhythmAnalyzer:
    @staticmethod
    def detect_bpm(audio_path: str) -> float:
        """用 librosa.beat.tempo() 检测 BPM，返回 tempo 值"""

    @staticmethod
    def get_beat_frames(audio_path: str, sr: int = 22050) -> list[float]:
        """返回节拍时间戳列表（秒），用于对齐镜头"""

    @staticmethod
    def is_strong_beat(beat_idx: int, time_sig_numerator: int = 4) -> bool:
        """判断是否为强拍（第一拍）"""

    @staticmethod
    def align_clip_to_beats(
        segment_durations: list[float],
        beats: list[float],
        tolerance: float = 0.15
    ) -> list[float]:
        """
        将每段时长微调对齐到最近的节拍
        - tolerance: 最大允许调整量（秒）
        - 只缩短或延长 2-3 帧，不改变文案完整性
        """
```

**集成方式：**
1. Pipeline `RENDER_VIDEO` 步骤开始时：
   - 检查 `visual.beat_sync` 是否启用
   - 如果启用 → 对 BGM 文件调用 `get_beat_frames()` → 暂存节拍时间戳
2. Renderer `_render_clips()` 中：
   - 如果有节拍时间戳 → 将转场时间微调到最近节拍
   - 强拍位置使用 `visual.strong_beat_transition`（如 circleopen）
   - 弱拍位置使用 `visual.weak_beat_transition`（如 dissolve）
3. BGM 无或配置关闭 → 降级到现有行为（基于持续时间的转场）

**影响范围：**
- 仅修改 BGM 音频分析的步骤，不影响无 BGM 视频
- 测试通过条件：无 BGM 时不触发 librosa（0 影响），有 BGM + 启用开关时对齐生效

**依赖安装：**
```batch
pip install librosa soundfile
```
`librosa` 依赖链（`numpy`, `scipy`, `pooch`, `decorator`, `lazy-loader`, `packaging`, etc.）总计约 30MB。`soundfile` 约 2MB。

## 风险与缓释

| 风险 | 影响 | 缓释 |
|------|------|------|
| librosa + scipy ≈ 50MB wheel 下载失败 | Phase 2D 打包卡住 | 惰性导入 + `beat_sync: false` 默认关闭 |
| 分类映射太主观，部分组合不常用 | 映射表可能没有实际使用 | 未知组合 fallback 到最邻近映射（知识→abstract、商业→product） |
| 语义扩展关键词过多干扰 | 原有精确匹配反向被稀释 | 扩展关键词单独评分，加权系数小（0.2） |
| Phase 1 文档未覆盖新异常类 | 异常消息变成英文 | 异常类只提供英文描述字段，UI 层按需包中文 |
