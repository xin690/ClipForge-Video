## 一、软件概述

### 1.1 核心理念

> **AI 负责规划，程序负责执行。**

ClipForge 不是 AI 视频生成模型，而是 **AI 自动剪辑引擎**。它接受脚本 JSON 作为输入，从本地素材库中匹配素材，通过规则引擎自动决策，最终用 FFmpeg 渲染出完整的视频。

### 1.2 系统流程

```
  ┌─────────── 输入 A：AI 规划（NEW）───────────┐
  │  主题文案 → AI 生成脚本 → AI 搜索关键词      │
  │      → 自动下载素材 → 分类归档 → 扫描入库    │
  └───────────────┬────────────────────────────┘
                  ↓
  ┌─────────── 输入 B：手动操作 ────────────────┐
  │         脚本 JSON + 本地素材库               │
  └───────────────┬────────────────────────────┘
                  ↓
       素材检索系统（本地标签匹配）
                  ↓
       规则引擎（字幕/转场/运镜规则）
                  ↓
       Timeline 时间轴生成
                  ↓
       音频生成（TTS 配音 → 旁白对齐 → BGM 混合 → loudnorm 归一化）
                  ↓
       视频渲染（镜头归一化 → concat demuxer 拼接）
                  ↓
       音视频 mux（-c:v copy -c:a aac）
                  ↓
       字幕生成 + 叠加（libass 优先 / Pillow 回退）
                  ↓
       输出 MP4
```

### 1.3 适用视频类型

**推荐：** 知识类、新闻类、口播类、电商类、混剪类、短视频矩阵

**不推荐：** AI 电影、长剧情视频、高级影视特效、AI 原创动画

## 四、脚本 JSON 格式（输入规范）

脚本是 ClipForge 的**唯一输入**，必须为 UTF-8 编码的 JSON 文件。

### 4.1 完整示例

```json
{
  "title": "减脂知识科普",
  "duration": 21,
  "style": "knowledge",
  "voice": "female_01",
  "bgm": "bgm_knowledge.mp3",
  "segments": [
    {
      "id": 1,
      "text": "很多人减脂失败，不是运动不够",
      "keywords": ["减脂", "健身"],
      "emotion": "normal",
      "duration": 7
    },
    {
      "id": 2,
      "text": "而是饮食错误！热量控制才是关键",
      "keywords": ["饮食", "热量"],
      "emotion": "strong",
      "duration": 7
    }
  ]
}
```

### 4.2 顶层字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `title` | string | **是** | — | 视频标题，也作为输出文件名 |
| `duration` | int | **是** | — | 总时长（秒），范围 1-3600 |
| `style` | string | 否 | `"knowledge"` | 视频风格：`knowledge` / `news` / `entertainment` / `commerce` |
| `voice` | string | 否 | `"female_01"` | 配音角色：`female_01` / `male_01` |
| `bgm` | string | 否 | `""` | BGM 文件名（需放在 `assets/bgm/` 下），留空则无背景音乐 |
| `segments` | array | **是** | — | 分段列表，最少 1 条 |

### 4.3 每个 segment 字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | int | **是** | — | 分段编号，从 1 开始递增 |
| `text` | string | **是** | — | 旁白文本，也是字幕文本（UTF-8 中文） |
| `keywords` | array[string] | 否 | `[]` | 关键词，用于匹配素材库中的视频 |
| `emotion` | string | 否 | `"normal"` | 情感：`normal` / `strong` / `happy` / `sad` / `calm` |
| `duration` | int | 否 | `5` | 本段时长（秒），范围 1-60 |

### 4.4 保存位置

脚本文件应放在 `scripts/` 目录下，UTF-8 编码（无 BOM）。

### 4.5 预置示例

软件自带 3 个示例脚本：

| 文件 | 风格 | 分段 | 总时长 | 说明 |
|------|------|------|--------|------|
| `scripts/sample_knowledge.json` | knowledge | 3 | 21s | 减脂知识科普 |
| `scripts/sample_commerce.json` | commerce | 3 | 15s | 商业趋势分析 |
| `scripts/sample_health.json` | entertainment | 3 | 18s | 健康生活小贴士 |
| `scripts/sample_nature.json` | knowledge | 3 | 24s | 中国自然风光 |

## 五、素材收集与配置

### 5.1 目录结构

所有素材放在 `assets/` 目录下，按类型分目录存放：

```
assets/
├── videos/           视频素材
│   ├── food_01.mp4
│   ├── fitness_01.mp4
│   └── ...
├── images/           图片素材（可选）
├── bgm/              背景音乐
│   ├── bgm_knowledge.mp3
│   ├── bgm_upbeat.mp3
│   └── ...
└── voice/            预设配音文件（可选，通常由 TTS 自动生成）
```

### 5.2 支持的素材格式

| 类型 | 支持的扩展名 |
|------|-------------|
| 视频 | `.mp4` `.mov` `.avi` `.mkv` `.webm` `.flv` |
| 图片 | `.jpg` `.jpeg` `.png` `.gif` `.bmp` `.webp` |
| BGM | `.mp3` `.wav` `.ogg` `.flac` `.aac` `.m4a` |
| 配音 | `.mp3` `.wav` `.ogg` `.m4a` |

### 5.3 标签自动提取规则

素材入库时，**自动从文件路径提取标签**（无需手动标注）：

```
assets/videos/food/healthy_salad.mp4
    → 文件名不含扩展名: "healthy_salad"
    → 按 - _ 空格拆分: ["healthy", "salad"]
    → 父目录名: "food"
    → 最终标签: ["healthy", "salad", "food"]
```

自动过滤的**停用词**：

**中文：** 的、了、是、在、有、和、就、不、人、都、一、一个、上、也、很、到、说、要、去、你、会、着、没有、看、好、自己、这

**英文：** the、a、an、in、on、at、to、for、of、with、and、or、is、are、was、were、be、been、being、have、has、had、do、does、did、will、would、can、could、may、might、shall、should、this、that、these、those、it、its、it's

### 5.4 素材匹配逻辑

Pipeline 运行时，对每个 segment 的 `keywords` 进行标签匹配（**v0.4.1 升级版**）：

1. **关键词评分**（[0,1] 归一化）：精确匹配 +1.0、同义词匹配 +0.6、模糊匹配 +0.3，除以关键词总数
2. **情绪基调评分**：素材标签与情绪色板重合度
3. **多样性评分**：避免连续重复使用同一素材
4. **分类匹配评分**（v0.4.1 升级）：style×emotion → 20 组中文+英文分类标签，通过 SynonymEngine 模糊匹配（阈值 ≥0.4）
5. **扩展评分**：从 TTS 文案自动提取的实体名词匹配
6. **候选池**：500 个（v0.4.1 从 100 提升），确保覆盖率
7. 非加权综合排序，取 TopK（默认 3）

**关键变化（v0.4.1）：** 即使关键词完全无匹配，素材仍可凭情绪/分类/多样性评分入选。避免因分词不精确导致的「零匹配」问题。

**匹配示例：**
```
segment keywords: ["减脂", "健身"]

素材标签 ["减脂", "沙拉", "低卡"] → kw=1.0/2=0.5 + 分类+情绪+多样性 → 总分  ← 选中
素材标签 ["健身", "跑步", "运动"] → kw=1.0/2=0.5 + 分类+情绪+多样性 → 总分
素材标签 ["饮食", "健康"]         → kw=0.0 + 但情绪/分类匹配可能入选
```

### 5.5 BGM 匹配规则

如果脚本指定了 `bgm` 文件名，直接使用该文件。
如果脚本未指定但设定了 `style`，规则引擎按风格推荐：

| 风格 | 推荐 BGM | 路径 |
|------|---------|------|
| `knowledge` | `bgm_knowledge.mp3` | `assets/bgm/bgm_knowledge.mp3` |
| `news` | `bgm_news.mp3` | `assets/bgm/bgm_news.mp3` |
| `entertainment` | `bgm_upbeat.mp3` | `assets/bgm/bgm_upbeat.mp3` |
| `commerce` | `bgm_commerce.mp3` | `assets/bgm/bgm_commerce.mp3` |

BGM 音量在 `config.yaml` 中设置：`bgm.volume: 0.3`（0.0~1.0），默认 30%。

### 5.6 素材入库流程

1. 将素材文件放入 `assets/videos/`、`assets/bgm/` 等对应目录
2. 启动 GUI → 点击菜单 "工具 → 扫描素材库"（快捷键 Ctrl+Shift+S）
3. 素材自动入库，出现在素材浏览器表格中
4. 也可从素材浏览器点击 "导入素材" 按钮选择文件

## 六、配置文件指南

配置文件 `config.yaml` 位于项目根目录。

### 6.1 完整配置

```yaml
app:
```

### 7.7 AI 视频规划（NEW）

AI 视频规划功能支持**从零创建视频**：只需输入主题文案，AI 自动生成脚本并下载素材。

---

#### 7.7.1 前提条件

**必须配置（在设置对话框或 `config.yaml`）：**

| 配置项 | 位置 | 说明 |
|--------|------|------|
| AI 启用 + API Key | 设置 → AI 标签页 | 支持 OpenAI / Qwen / DeepSeek |
| 素材下载 API Key | 设置 → 素材下载标签页 | Pexels Key + Pixabay Key 独立设置（至少填一个，可同时搜索两平台） |

> **AI 规划与素材下载可分别独立使用**：
> - 仅启用 AI → 可生成脚本 JSON，但无法下载素材（使用本地素材库匹配）
> - 仅启用下载器 → 可手动搜索下载素材，供手动脚本使用

---

#### 7.7.2 操作步骤

**第 1 步：打开 AI 规划对话框**

在「脚本编辑」标签页顶部工具栏，点击 **「AI 规划」** 按钮。

**第 2 步：输入主题文案**

在「主题文案」文本框中输入视频主题，越详细效果越好：

```
示例 1：「制作一个关于中国自然风光的短视频，展示高山、大海、草原」
示例 2：「做一个减脂知识科普视频，包含饮食和运动两方面的建议」
示例 3：「介绍 iPhone 15 Pro 的钛金属边框和 A17 Pro 芯片」
```

**第 3 步：选择视频风格**

从下拉框中选择视频风格：

| 风格 | 适用场景 | 语气特点 | 素材风格 |
|------|----------|----------|----------|
| knowledge | 知识科普 | 专业、客观 | 纪录片风、实拍 |
| news | 新闻资讯 | 简洁、严肃 | 新闻画面、商务 |
| entertainment | 娱乐 | 生动、有趣 | 多彩、活泼 |
| commerce | 电商带货 | 直接、有说服力 | 产品展示、特写 |

**第 4 步：点击「AI 规划」**

点击后对话框进入加载状态，AI 将在后台调用（通过 QThread 异步，不阻塞 UI）：

- 等待数秒后，预览表格将显示 AI 生成的分段内容
- 每个分段展示：编号、口播文案、中文关键词、情绪、时长
- 状态栏显示分段数和搜索关键词数

> **AI 输出质量取决于 LLM 模型**：推荐使用 `deepseek-chat` 或 `gpt-4o` 获得更准确的结果。

**第 5 步：预览与确认**

在预览表格中检查 AI 生成的内容：

- **口播文案**：可直接修改（编辑后不影响搜索关键词）
- **关键词**：影响素材匹配精度，确保与目标素材的标签有重叠
- **情绪**：决定字幕样式和默认运镜（normal=蓝色/static，strong=红色/zoom，calm=浅蓝/zoom）
- **时长**：4-8 秒为宜，总时长自动累加

切换到 **「内容检查」** 标签页查看自动质检结果：

- 10 项检查：段落数、时长、文案长度、关键词数量、情绪有效性、情绪序列、关键词多样性、配音角色、风格、BGM、总时长一致性
- 结果标注通过 ✅ / 警告 ⚠️ / 失败 ❌
- 支持预设切换：tiktok / youtube / commerce（不同平台有独立阈值）
- 点击「重新检查」可在编辑后刷新结果

> 如果对结果不满意，可修改主题文案后重新点击「AI 规划」。

**第 6 步（可选）：下载素材**

点击 **「下载素材」** 按钮，程序将：

1. 为每个分段使用 AI 生成的英文搜索词（v0.4.1 自动扩展为 2-3 个搜索变体，例：mountain → mountain documentary / mountain nature）
2. 同时调用 Pexels + Pixabay API 搜索匹配视频（若两个 Key 均已配置），去重后按分辨率降序排序
3. 自动选择最高画质文件下载，下载后自动将 API 标签合并到数据库
4. 保存到 `assets/videos/`，文件名包含分段编号
5. 下载完成后自动增量扫描入库

进度条显示下载状态，单个文件下载失败不影响其他文件。

> **跳过下载**：如果已有本地素材库，直接保存脚本即可，渲染时使用本地 Matcher 匹配。

**第 7 步：保存脚本**

点击 **「保存脚本」** 按钮：

- 自动保存为 `scripts/{标题}_{风格}.json`
- 使用 UTF-8 编码，Pydantic 校验确保格式正确
- 保存后在脚本编辑器中打开即可编辑

**第 8 步：生成视频**

回到主界面流程：

1. 在脚本编辑器中打开刚保存的 JSON
2. 点击「工具 → 扫描素材库」（建议增量扫描）
3. 点击「▶ 生成视频」启动渲染管线

## 八、输出规格

### 8.1 视频文件规格

| 项目 | 规格 |
|------|------|
| 容器格式 | MP4 |
| 视频编码 | H.264 (libx264)，`-pix_fmt yuv420p` |
| 音频编码 | AAC (128kbps)，`-ar 44100 -ac 2`（立体声） |
| 分辨率 | 默认 1920×1080（可配置） |
| 帧率 | 默认 30fps（`-vf fps=30` 滤镜级转换） |
| CRF | 默认 23（可配置） |
| 编码预设 | 默认 veryfast（可配置） |
| GOP 设置 | `-g 60 -keyint_min 60 -sc_threshold 0`（保证拼接无缝） |
| 优化参数 | `-video_track_timescale 30000 -movflags +faststart` |

### 8.2 音频处理（视频+音频分离管线）

| 阶段 | 处理方式 |
|------|---------|
| 镜头编码 | 统一使用 `-an`（无音频输出），避免含音频镜头导致 concat 丢帧 |
| TTS 配音 | edge-tts 生成 WAV，通过 `adelay{ms}:all=1` + `atrim=end=duration` 定位到正确时间点 |
| 旁白混合 | 多个配音文件通过 `amix=inputs=n:duration=longest:normalize=0` 混合 |
| BGM 混合 | 自动循环（`aloop=size=1000000000`）+ `volume={bgm_volume}` 音量控制 + `amix=duration=longest` |
| 响度归一化 | EBU R128 标准：`loudnorm=I=-16:TP=-1.5:LRA=11` |
| 音视频 mux | 视频 `-c:v copy`（无重新编码）+ 音频 `-c:a aac -ar 44100 -ac 2` |

### 8.3 转场效果

| 转场类型 | 实现方式 | 触发条件 |
|---------|---------|---------|
| cut（硬切） | concat demuxer（`-f concat`） | 当前使用（所有镜头合并为视频序列后统一 concat） |
| fade（淡入淡出） | xfade=transition=fade（需镜头含音频，当前默认跳过） | 3 秒 < 镜头时长 ≤ 8 秒 |
| slide（滑动） | xfade=transition=slideleft（需镜头含音频，当前默认跳过） | 镜头时长 > 8 秒 |

**说明：** 当前管线中所有镜头编码时使用 `-an`（无音频输出），因此 xfade 的 `acrossfade` 无法使用。转场默认使用 concat demuxer 直切模式。如需要淡入淡出或滑动转场，需确保镜头包含音频轨道。

### 8.5.1 v0.4.0 视觉增强（画面 + 声音 + 节奏）

#### 情绪感知调色

渲染时根据每段 emotion 自动应用段级调色滤镜链：

| emotion | 效果 | 滤镜参数 |
|---------|------|---------|
| normal | 微提亮 + 微对比 | `eq=brightness=0.02:contrast=1.05` |

| calm | 提亮 + 柔和 + gamma | `eq=brightness=0.03:contrast=0.95:saturation=0.85:gamma=1.05` |
| happy | 明亮 + 高饱和 | `eq=brightness=0.05:contrast=1.05:saturation=1.25:gamma=1.02` |
| sad | 暗色 + 低饱和 | `eq=brightness=-0.03:contrast=0.90:saturation=0.70` |

滤镜链顺序：`运镜滤镜 → 风格调色(COLOR_PRESETS) → 情绪调色(EMOTION_GRADING) → LUT → format=yuv420p`

**LUT 支持**：配置 `visual.color.lut_path` 指向 `.cube` 文件目录，渲染时通过 `lut3d` 滤镜自动加载。

**Ken Burns 增强**：`visual.camera.zoom_speed`（默认 0.0015）和 `visual.camera.max_zoom`（默认 1.5）控制缩放速度与最大放大比例。

#### 节拍感知转场（beat sync）

启用于 `visual.beat_sync: true`（默认关闭）：

1. 渲染开始时，对 BGM 文件调用 librosa 检测 BPM 和节拍时间戳
2. 每个镜头开始时，查找最近的节拍位置
3. 强拍（每小节第 1 拍）→ 自动使用 `visual.strong_beat_transition`（默认 circleopen）
4. 弱拍 → 自动使用 `visual.weak_beat_transition`（默认 dissolve）
5. 无 BGM 或开关关闭 → 降级到预设转场

**新增依赖**：`librosa>=0.10.0` + `soundfile>=0.12.0`（仅在启用 beat_sync 时加载）

#### 内容探针（content analysis）

启用于 `scanner.content_analysis: true`（**v0.4.1 默认启用**）：

扫描素材时对每个文件调用 ffprobe 提取内容属性，自动追加标签：
- **分辨率**：`res:4k` / `res:1080p` / `res:720p` 等
- **帧率/动态**：`motion:smooth`（50fps+）/ `motion:standard`（25fps+）/ `motion:cinematic`（20fps+）
- **色彩**：`color:hdr`（10bit）/ `color:full`（yuv444）
- **时长**：`duration:short`（≤5s）/ `duration:medium`（5-15s）/ `duration:long`（15s+）
- **音频**：`audio:hires` / `audio:stereo` / `audio:lossless` 等

**v0.4.1 新增 3 种探针**：

| 探针 | 方法 | 标签 | 用途 |
|------|------|------|------|
| **场景变化** | ffprobe 取 50 帧分析 I 帧比 | `scene:high`(>30%) / `scene:low`(<10%) / `scene:medium` + `dynamic`/`static` | 高动态镜头配 strong 情绪，静态配 calm |
| **主色检测** | ffmpeg 取 1 帧 + Pillow HSV 桶统计 | `color:red` / `color:green` / `color:blue` / `color:bright` / `color:dark` 等 | 按情绪选择色彩基调 |
| **黑边检测** | ffmpeg cropdetect 2 秒分析 | `letterbox` + `aspect:ultrawide`/`standard` | 过滤黑边素材，避免画幅不一致 |

标签会参与素材匹配评分（通过分类映射和关键词扩展），无需任何手动标注。默认启用后，扫描时间约增加 ~3 秒/视频。

#### 语义匹配升级

**v0.4.0**：新增非对称加权 10 因子综合评分：

```
总分 = keyword_score × w_kw × 100
     + emotion_tone_score × w_tone × 100
     + diversity_score × w_div × 100
     + category_match_score × 20    ← ☆v0.4.0 新增
     + expand_score × 10            ← ☆v0.4.0 新增
```

- **category_match_score**：20 组 style×emotion → 视觉标签映射（如（knowledge, strong）→ tech/dynamic/data/blueprint），与素材标签交集比例 ×20
- **expand_score**：从 TTS 文案自动提取的实体名词（正则中文 2-gram），与素材标签同义词模糊匹配得分 ×10
- 匹配低分（< `visual.semantic.min_confidence`）→ 日志 warning，但不中断流程

**v0.4.1 优化**：

- **关键词评分 [0,1] 归一化**：精确=1.0、同义词=0.6、模糊=0.3，除以关键词总数（F10）
- **零匹配不限入**：`kw_score <= 0` 不再跳过素材，即关键词无匹配时仍可凭情绪/分类/多样性评分入选（F1）
- **分类匹配同义词化**：`category_match_score` 使用 SynonymEngine 模糊匹配，阈值 ≥0.4（F4）
- **中文分类标签**：20 组 CATEGORY_MAP 各补 5 个中文标签，支持中文关键词直接匹配（F3）
- **候选池 100→500**：避免早期截断遗漏匹配素材（F7）

```
示例：output/减脂知识科普_knowledge.mp4
```

## 九、常见问题

### Q1: 启动时提示 FFmpeg 未找到

确保 FFmpeg 已安装并加入 PATH：
```bash
ffmpeg -version
```
如果命令不可用，从 https://www.gyan.dev/ffmpeg/builds/ 下载 ffmpeg-release-full.7z，解压后将其 `bin` 目录加入系统 PATH。

### Q2: 输出视频没有字幕

可能原因：
1. 脚本中 segment 的 `text` 字段为空 → 检查脚本
2. FFmpeg 没有 libass → 命令行运行 `py -c "from core.ffmpeg import has_libass; print(has_libass())"` 检查
3. 回退 Pillow 叠加需要安装 Pillow → `py -m pip install Pillow`

### Q3: TTS 配音没有声音

1. 检查网络（edge-tts 需要联网）→ 离线时自动回退到 pyttsx3（Windows SAPI5）
2. 检查日志：是否回退到 pyttsx3 或静音音频
3. TTS 速率无效时检查：速率参数始终包含正负号（如 `+0%`、`+25%`），早期版本可能在此处报错
4. 删除 `cache/tts/` 目录（旧缓存可能含静音文件），重试渲染

### Q4: 如何调整 BGM 音量

编辑 `config.yaml`，修改：
```yaml
bgm:
  volume: 0.3    # 改为 0.5 增大音量
```

### Q5: 渲染时间太长

1. 在设置中调整编码预设为 `ultrafast`
2. 降低 CRF 到 28
3. 减少分段数量或缩短总时长

### Q6: 素材匹配结果不理想

1. 确保素材文件名包含有意义的标签（如 `healthy_salad.mp4` 优于 `video001.mp4`），标签按 `-`、`_`、空格分割
2. 在脚本 segment 的 `keywords` 中填写准确的关键词
3. 将素材放入有意义的子目录中（如 `assets/videos/food/`），目录名自动作为标签
4. **v0.4.1 优化**：即使 keywords 完全无匹配，素材仍会基于情绪/分类/多样性评分入选；中英文关键词均可匹配（CATEGORY_MAP 含 100 个中文分类标签）；同义词匹配已扩展至分类匹配

### Q7: 如何中途取消渲染

1. 生成过程中点击「取消」按钮 → FFmpeg 进程立即终止
2. 管线中的 `cancel_event`（threading.Event）从 UI 传播到 Pipeline → Renderer → FFmpeg execute
3. 取消后临时文件自动清理（`_cleanup_temp()` 删除 `_temp_voice` 和 `_temp_subtitle.ass`）

### Q8: 程序异常退出 / QThread 崩溃

原因：旧版在窗口关闭时可能出现 "Destroyed while thread is still running" 错误。
修复：v0.2.0 中所有工作线程使用 `_worker_cleanup()` 方法（`wait()` + `deleteLater()`），确保线程安全退出。

### Q9: Edge-TTS 报 "Invalid rate '0%'"

原因：旧版在 `speed=1.0` 时生成速率 `0%`（缺少符号前缀），edge-tts 要求必须有 `+` 或 `-` 符号。
修复：v0.2.0 使用 `f"{int(...):+d}%"` 格式，始终包含正负号（产生 `"+0%"`）。

### Q10: AI 规划失败怎么办

1. 检查 `ai.enabled` 是否为 `true`，`ai.api_key` 是否正确填入
2. 检查网络连接（AI 规划需要访问 LLM API）
3. 检查 API 余额是否充足
4. 查看控制台日志（`logging.level: DEBUG` 可查看详细调试信息）
5. 可以多次重试（修改主题文案后重新点击「AI 规划」）
6. 如果 AI 持续返回格式错误，尝试更换模型（如 `gpt-4o` 比 `gpt-4o-mini` 更可靠）

### Q11: 素材下载失败？

1. 检查 `downloader.pexels_key` 或 `downloader.pixabay_key` 是否配置（v0.4.1 支持双 Key 独立设置）
2. Pexels 免费注册：https://www.pexels.com/api/（200次/小时）
3. Pixabay 免费注册：https://pixabay.com/api/docs/（5000次/月）
4. 同时配置两个 Key 可并行搜索两平台，增加命中率
5. Pexels 限速 200 次/小时，超过后返回 429
6. 检查网络连接（下载需要访问 CDN）
7. 部分搜索关键词可能无结果（v0.4.1 自动生成 2-3 个搜索变体提高命中率）
8. 下载失败不会阻止其他文件的下载，单个文件失败可忽略

### Q12: 希望只使用 AI 规划（不下载素材）？

完全可以。在 AI 规划对话框中跳过「下载素材」步骤，直接「保存脚本」。后续渲染管线使用本地素材库匹配。

### Q13: AI 规划的脚本能否手动修改？

可以。保存后的脚本 JSON 可在脚本编辑器中正常编辑，修改文案、关键词、时长等参数后重新生成视频。
