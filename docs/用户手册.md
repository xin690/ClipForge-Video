## 一、软件概述

### 1.1 核心理念

> **AI 负责规划，程序负责执行。**

ClipForge 不是 AI 视频生成模型，而是 **AI 自动剪辑引擎**。它接受脚本 JSON 作为输入，从本地素材库中匹配素材，通过规则引擎自动决策，最终用 FFmpeg 渲染出完整的视频。

### 1.2 系统流程

```
输入脚本 JSON
    ↓
素材检索系统（本地标签匹配）
    ↓
规则引擎（字幕/转场/运镜规则）
    ↓
AI 镜头规划（可选，默认关闭）
    ↓
Timeline 时间轴生成
    ↓
音频生成（TTS 配音 → 旁白对齐 → BGM 混合 → loudnorm 归一化）
    ↓
视频渲染（镜头归一化 → concat demuxer 拼接）
    ↓
音视频 mux（-c:v copy -c:a aac）
    ↓
字幕生成 + 叠加（libass 优先 / Pillow 回退）
    ↓
输出 MP4
```

### 1.3 适用视频类型

**推荐：** 知识类、新闻类、口播类、电商类、混剪类、短视频矩阵

**不推荐：** AI 电影、长剧情视频、高级影视特效、AI 原创动画

## 二、功能全览

### 2.1 13 个核心模块

| 模块 | 文件 | 功能 |
|------|------|------|
| 配置管理 | `core/config.py` | YAML 配置加载/保存/热读，支持点号路径查询，环境变量覆盖 |
| 数据模型 | `core/models.py` | 6 个 Pydantic 模型：`Script`、`Segment`、`Asset`、`Timeline`、`TimelineItem`、`Project` |
| 数据库 | `core/database.py` | SQLite 素材库 CRUD、标签搜索、统计查询、脚本历史记录 |
| 素材扫描 | `core/scanner.py` | 全量/增量扫描 assets/ 目录，ffprobe 提取视频/音频元数据，文件名自动提取标签 |
| 素材匹配 | `core/matcher.py` | 关键词标签匹配 + 同义词扩展 + 评分排序，支持视频/BGM 按类型过滤 |
| 规则引擎 | `core/rules.py` | 4 条内置规则：字幕样式（emotion → 4 种样式）、转场（时长 → cut/fade/slide）、运镜（风格 → static/slow_zoom/pan）、时长调整 |
| 时间轴构建 | `core/timeline.py` | 脚本 → Timeline 自动构建 + 校验（时间连续性、素材存在性） |
| 语音合成 | `core/tts.py` | 三引擎降级：edge-tts（在线）→ pyttsx3（离线）→ FFmpeg 静音音频，MD5 缓存 |
| 字幕生成 | `core/subtitle.py` | 文本 → SRT/ASS 字幕，4 种样式（Microsoft YaHei 字体），Whisper 音频识别（可选） |
| FFmpeg 封装 | `core/ffmpeg.py` | FFmpegBuilder 链式构建器、FFmpeg 可用性检测（60s TTL 缓存）、线程化命令执行（含 progress/cancel）、concat demuxer、concat filter、**xfade 转场**、**loudnorm 响度归一化** |
| 视频渲染 | `core/renderer.py` | 视频+音频分离管线：镜头归一化(-an, -vf fps=30, yuv420p) → concat demuxer 视频拼接 → 旁白对齐(adelay+atrim+amix) → BGM 混合(volume+amix) → loudnorm 归一化 → mux(-c:v copy -c:a aac) → 硬字幕叠加（libass 优先 / Pillow 回退）→ **运镜效果（zoompan/crop）** |
| 工作流管线 | `core/pipeline.py` | 完整管线编排（8 步），支持进度回调、取消、批量处理、脚本 JSON 校验 |
| AI 规划 | `core/ai_planner.py` | OpenAI/Qwen/DeepSeek API 可选集成（默认关闭），负责素材选择 / 文案润色 / 时长建议 |

## 四、脚本 JSON 格式（输入规范）

脚本是 ClipForge 的**唯一输入**，必须为 UTF-8 编码的 JSON 文件。

### 4.1 完整示例

```json
{
  "title": "减脂知识科普",
  "duration": 21,
  "style": "knowledge",
  "voice": "female_01",
  "bgm": "bgm_knowledge.mp3",
  "segments": [
    {
      "id": 1,
      "text": "很多人减脂失败，不是运动不够",
      "keywords": ["减脂", "健身"],
      "emotion": "normal",
      "duration": 7
    },
    {
      "id": 2,
      "text": "而是饮食错误！热量控制才是关键",
      "keywords": ["饮食", "热量"],
      "emotion": "strong",
      "duration": 7
    }
  ]
}
```

### 4.2 顶层字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `title` | string | **是** | — | 视频标题，也作为输出文件名 |
| `duration` | int | **是** | — | 总时长（秒），范围 1-3600 |
| `style` | string | 否 | `"knowledge"` | 视频风格：`knowledge` / `news` / `entertainment` / `commerce` |
| `voice` | string | 否 | `"female_01"` | 配音角色：`female_01` / `male_01` |
| `bgm` | string | 否 | `""` | BGM 文件名（需放在 `assets/bgm/` 下），留空则无背景音乐 |
| `segments` | array | **是** | — | 分段列表，最少 1 条 |

### 4.3 每个 segment 字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | int | **是** | — | 分段编号，从 1 开始递增 |
| `text` | string | **是** | — | 旁白文本，也是字幕文本（UTF-8 中文） |
| `keywords` | array[string] | 否 | `[]` | 关键词，用于匹配素材库中的视频 |
| `emotion` | string | 否 | `"normal"` | 情感：`normal` / `strong` / `happy` / `sad` / `calm` |
| `duration` | int | 否 | `5` | 本段时长（秒），范围 1-60 |

### 4.4 保存位置

脚本文件应放在 `scripts/` 目录下，UTF-8 编码（无 BOM）。

### 4.5 预置示例

软件自带 3 个示例脚本：

| 文件 | 风格 | 分段 | 总时长 | 说明 |
|------|------|------|--------|------|
| `scripts/sample_knowledge.json` | knowledge | 3 | 21s | 减脂知识科普 |
| `scripts/sample_commerce.json` | commerce | 3 | 15s | 商业趋势分析 |
| `scripts/sample_health.json` | entertainment | 3 | 18s | 健康生活小贴士 |
| `scripts/sample_nature.json` | knowledge | 3 | 24s | 中国自然风光 |

## 五、素材收集与配置

### 5.1 目录结构

所有素材放在 `assets/` 目录下，按类型分目录存放：

```
assets/
├── videos/           视频素材
│   ├── food_01.mp4
│   ├── fitness_01.mp4
│   └── ...
├── images/           图片素材（可选）
├── bgm/              背景音乐
│   ├── bgm_knowledge.mp3
│   ├── bgm_upbeat.mp3
│   └── ...
└── voice/            预设配音文件（可选，通常由 TTS 自动生成）
```

### 5.2 支持的素材格式

| 类型 | 支持的扩展名 |
|------|-------------|
| 视频 | `.mp4` `.mov` `.avi` `.mkv` `.webm` `.flv` |
| 图片 | `.jpg` `.jpeg` `.png` `.gif` `.bmp` `.webp` |
| BGM | `.mp3` `.wav` `.ogg` `.flac` `.aac` `.m4a` |
| 配音 | `.mp3` `.wav` `.ogg` `.m4a` |

### 5.3 标签自动提取规则

素材入库时，**自动从文件路径提取标签**（无需手动标注）：

```
assets/videos/food/healthy_salad.mp4
    → 文件名不含扩展名: "healthy_salad"
    → 按 - _ 空格拆分: ["healthy", "salad"]
    → 父目录名: "food"
    → 最终标签: ["healthy", "salad", "food"]
```

自动过滤的**停用词**：

**中文：** 的、了、是、在、有、和、就、不、人、都、一、一个、上、也、很、到、说、要、去、你、会、着、没有、看、好、自己、这

**英文：** the、a、an、in、on、at、to、for、of、with、and、or、is、are、was、were、be、been、being、have、has、had、do、does、did、will、would、can、could、may、might、shall、should、this、that、these、those、it、its、it's

### 5.4 素材匹配逻辑

Pipeline 运行时，对每个 segment 的 `keywords` 进行标签匹配：

1. **精确匹配**权重 2.0：segment 关键词直接命中素材标签
2. **同义词匹配**权重 1.0：关键词通过同义词表扩展后命中标签（需配置同义词表）
3. 按评分排序，取 TopK（默认 3）
4. 取最高分素材作为当前镜头

**匹配示例：**
```
segment keywords: ["减脂", "健身"]

素材标签 ["减脂", "沙拉", "低卡"] → 得分 2.0/2 = 1.0  ← 选中
素材标签 ["健身", "跑步", "运动"] → 得分 2.0/2 = 1.0
素材标签 ["饮食", "健康"]         → 得分 0.0/2 = 0.0
```

### 5.5 BGM 匹配规则

如果脚本指定了 `bgm` 文件名，直接使用该文件。
如果脚本未指定但设定了 `style`，规则引擎按风格推荐：

| 风格 | 推荐 BGM | 路径 |
|------|---------|------|
| `knowledge` | `bgm_knowledge.mp3` | `assets/bgm/bgm_knowledge.mp3` |
| `news` | `bgm_news.mp3` | `assets/bgm/bgm_news.mp3` |
| `entertainment` | `bgm_upbeat.mp3` | `assets/bgm/bgm_upbeat.mp3` |
| `commerce` | `bgm_commerce.mp3` | `assets/bgm/bgm_commerce.mp3` |

BGM 音量在 `config.yaml` 中设置：`bgm.volume: 0.3`（0.0~1.0），默认 30%。

### 5.6 素材入库流程

1. 将素材文件放入 `assets/videos/`、`assets/bgm/` 等对应目录
2. 启动 GUI → 点击菜单 "工具 → 扫描素材库"（快捷键 Ctrl+Shift+S）
3. 素材自动入库，出现在素材浏览器表格中
4. 也可从素材浏览器点击 "导入素材" 按钮选择文件

| 网站 | 类型 | 许可证 |

## 六、配置文件指南

配置文件 `config.yaml` 位于项目根目录。

### 6.1 完整配置

```yaml
app:
  name: ClipForge
  version: 0.2.0

paths:
  assets: ./assets              # 素材目录
  scripts: ./scripts            # 脚本目录（GUI 默认打开位置）
  output: ./output              # 输出视频目录
  cache: ./cache                # 缓存（TTS/字幕/渲染临时文件）
  database: ./database/clipforge.db  # SQLite 数据库文件

video:
  width: 1920                   # 输出分辨率宽
  height: 1080                  # 输出分辨率高
  fps: 30                       # 帧率
  preset: veryfast              # x264 编码预设（ultrafast~slow）
  crf: 23                       # 视频质量（0-51，越小越好）

tts:
  engine: edge-tts              # TTS 引擎：edge-tts / piper
  voice: zh-CN-XiaoxiaoNeural   # 中文语音
  speed: 1.0                    # 语速（0.5-2.0）

subtitle:
  engine: text                  # 字幕模式：text（文本转）/ whisper（音频识别）
  whisper_model: tiny           # Whisper 模型：tiny / base / small
  device: cpu                   # 运行设备：cpu / cuda

ai:
  enabled: false                # 是否启用 AI 规划（默认关闭）
  provider: openai              # API 提供商：openai / qwen / deepseek
  api_key: ""                   # API Key
  model: gpt-4o-mini            # 模型名称
  max_tokens: 500               # 每次调用的最大 token 数

bgm:
  volume: 0.3                   # BGM 相对音量（0.0-1.0）

logging:
  level: INFO                   # 日志级别：DEBUG / INFO / WARNING / ERROR
```

### 6.2 常用配置调整

| 需求 | 修改项 | 推荐值 |
|------|--------|--------|
| 降低输出文件大小 | `video.crf` | 28 |
| 提高渲染速度 | `video.preset` | ultrafast |
| 提高画质 | `video.crf` | 18 |
| 更改默认分辨率 | `video.width` / `video.height` | 1920 / 1080（横屏）或 1080 / 1920（竖屏） |
| 增大 BGM 音量 | `bgm.volume` | 0.5 |
| 启用 AI 规划 | `ai.enabled` | `true` + 填入 `api_key` |

### 7.2 脚本编辑标签页

1. **新建脚本**：点击 "新建" → 在 JSON 编辑区或表单中填写内容
2. **打开脚本**：点击 "打开" → 选择 `scripts/` 下的 .json 文件
3. **编辑方式**：
   - **JSON 模式**：直接编辑 JSON 文本（带语法高亮）
   - **表单模式**：在右侧表格中逐行编辑分段参数
4. **验证**：点击 "验证" 检查 JSON 格式和字段合法性
5. **生成视频**：点击 "▶ 生成视频" 启动完整管线
6. **预览时间轴**：点击 "预览" 查看镜头时间轴

### 7.3 素材管理标签页

1. **扫描素材库**：点击 "扫描素材库" 或菜单 "工具 → 扫描素材库"
2. **搜索素材**：在搜索框输入关键词，表格自动过滤
3. **类型过滤**：使用下拉框选择全部/视频/BGM/图片/配音
4. **导入素材**：点击 "导入素材" 选择文件
5. **删除素材**：选中行 → 点击 "删除选中"
6. **排序**：点击列标题排序

### 7.4 时间轴预览标签页

显示当前脚本的色彩块时间轴：
- 每个镜头为彩色圆角矩形
- 颜色按情感区分（normal=蓝, strong=红, big_yellow=黄, soft_white=灰, bold=粉）
- 矩形内显示字幕文本（截断至 12 字）+ 时间范围
- 顶部的刻度线标记秒数
- 等比例缩放（默认 60px/秒）

### 7.5 视频预览标签页

- 播放/暂停视频
- 拖拽进度条跳转
- 音量控制
- 时间显示（当前时间 / 总时长）

## 八、输出规格

### 8.1 视频文件规格

| 项目 | 规格 |
|------|------|
| 容器格式 | MP4 |
| 视频编码 | H.264 (libx264)，`-pix_fmt yuv420p` |
| 音频编码 | AAC (128kbps)，`-ar 44100 -ac 2`（立体声） |
| 分辨率 | 默认 1920×1080（可配置） |
| 帧率 | 默认 30fps（`-vf fps=30` 滤镜级转换） |
| CRF | 默认 23（可配置） |
| 编码预设 | 默认 veryfast（可配置） |
| GOP 设置 | `-g 60 -keyint_min 60 -sc_threshold 0`（保证拼接无缝） |
| 优化参数 | `-video_track_timescale 30000 -movflags +faststart` |

### 8.2 音频处理（视频+音频分离管线）

| 阶段 | 处理方式 |
|------|---------|
| 镜头编码 | 统一使用 `-an`（无音频输出），避免含音频镜头导致 concat 丢帧 |
| TTS 配音 | edge-tts 生成 WAV，通过 `adelay{ms}:all=1` + `atrim=end=duration` 定位到正确时间点 |
| 旁白混合 | 多个配音文件通过 `amix=inputs=n:duration=longest:normalize=0` 混合 |
| BGM 混合 | 自动循环（`aloop=size=1000000000`）+ `volume={bgm_volume}` 音量控制 + `amix=duration=longest` |
| 响度归一化 | EBU R128 标准：`loudnorm=I=-16:TP=-1.5:LRA=11` |
| 音视频 mux | 视频 `-c:v copy`（无重新编码）+ 音频 `-c:a aac -ar 44100 -ac 2` |

### 8.3 转场效果

| 转场类型 | 实现方式 | 触发条件 |
|---------|---------|---------|
| cut（硬切） | concat demuxer（`-f concat`） | 当前使用（所有镜头合并为视频序列后统一 concat） |
| fade（淡入淡出） | xfade=transition=fade（需镜头含音频，当前默认跳过） | 3 秒 < 镜头时长 ≤ 8 秒 |
| slide（滑动） | xfade=transition=slideleft（需镜头含音频，当前默认跳过） | 镜头时长 > 8 秒 |

**说明：** 当前管线中所有镜头编码时使用 `-an`（无音频输出），因此 xfade 的 `acrossfade` 无法使用。转场默认使用 concat demuxer 直切模式。如需要淡入淡出或滑动转场，需确保镜头包含音频轨道。

### 8.4 运镜效果

| 运镜类型 | 实现方式 | 触发条件 |
|---------|---------|---------|
| static（静止） | scale + pad 居中 | 风格为 news |
| slow_zoom（慢推） | zoompan 渐进放大 | 风格为 knowledge / commerce |
| pan（平移） | crop 左右移动 | 风格为 entertainment |

### 8.5 字幕规格

| 样式 | 字体 | 字号 | 颜色 | 适用情感 |
|------|------|------|------|---------|
| normal | Microsoft YaHei | 36 | 白色 | normal、默认 |
| big_yellow | Microsoft YaHei Bold | 48 | 金色 #FFD700 | strong |
| soft_white | Microsoft YaHei | 36 | 浅灰 #E0E0E0 | sad、calm |
| bold | Microsoft YaHei Bold | 42 | 粉色 #FF6B6B | happy |

字幕叠加方式：libass `ass` 滤镜（优先，需 FFmpeg 编译 libass）→ Pillow PNG overlay（回退）

## 九、常见问题

### Q1: 启动时提示 FFmpeg 未找到

确保 FFmpeg 已安装并加入 PATH：
```bash
ffmpeg -version
```
如果命令不可用，从 https://www.gyan.dev/ffmpeg/builds/ 下载 ffmpeg-release-full.7z，解压后将其 `bin` 目录加入系统 PATH。

### Q2: 输出视频没有字幕

可能原因：
1. 脚本中 segment 的 `text` 字段为空 → 检查脚本
2. FFmpeg 没有 libass → 命令行运行 `py -c "from core.ffmpeg import has_libass; print(has_libass())"` 检查
3. 回退 Pillow 叠加需要安装 Pillow → `py -m pip install Pillow`

### Q3: TTS 配音没有声音

1. 检查网络（edge-tts 需要联网）→ 离线时自动回退到 pyttsx3（Windows SAPI5）
2. 检查日志：是否回退到 pyttsx3 或静音音频
3. TTS 速率无效时检查：速率参数始终包含正负号（如 `+0%`、`+25%`），早期版本可能在此处报错
4. 删除 `cache/tts/` 目录（旧缓存可能含静音文件），重试渲染

### Q4: 如何调整 BGM 音量

编辑 `config.yaml`，修改：
```yaml
bgm:
  volume: 0.3    # 改为 0.5 增大音量
```

### Q5: 渲染时间太长

1. 在设置中调整编码预设为 `ultrafast`
2. 降低 CRF 到 28
3. 减少分段数量或缩短总时长

### Q6: 素材匹配结果不理想

1. 确保素材文件名包含有意义的标签（如 `healthy_salad.mp4` 优于 `video001.mp4`），标签按 `-`、`_`、空格分割
2. 在脚本 segment 的 `keywords` 中填写准确的关键词
3. 将素材放入有意义的子目录中（如 `assets/videos/food/`），目录名自动作为标签

### Q7: 如何中途取消渲染

1. 生成过程中点击「取消」按钮 → FFmpeg 进程立即终止
2. 管线中的 `cancel_event`（threading.Event）从 UI 传播到 Pipeline → Renderer → FFmpeg execute
3. 取消后临时文件自动清理（`_cleanup_temp()` 删除 `_temp_voice` 和 `_temp_subtitle.ass`）

### Q8: 程序异常退出 / QThread 崩溃

原因：旧版在窗口关闭时可能出现 "Destroyed while thread is still running" 错误。
修复：v0.2.0 中所有工作线程使用 `_worker_cleanup()` 方法（`wait()` + `deleteLater()`），确保线程安全退出。

### Q9: Edge-TTS 报 "Invalid rate '0%'"

原因：旧版在 `speed=1.0` 时生成速率 `0%`（缺少符号前缀），edge-tts 要求必须有 `+` 或 `-` 符号。
修复：v0.2.0 使用 `f"{int(...):+d}%"` 格式，始终包含正负号（产生 `"+0%"`）。
