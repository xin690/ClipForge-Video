# 示例：中国自然风光视频制作

## 零、标签提取规则（必读）

文件 `core/scanner.py` 的标签逻辑：

1. **文件名**：按 `-` `_` 空格分割，每段长度 >1 为标签
   - `雪山-云海.mp4` → `["雪山", "云海"]`
   - `雪山云海.mp4`  → `["雪山云海"]`（不分词！）
2. **父目录**：如果目录名不是 `videos/images/bgm/voice`，追加为标签
   - `高山/雪山-云海.mp4` → 标签 `["雪山", "云海", "高山"]`
3. **匹配**：`_score()` 精确匹配标签，命中权重 2.0，同义词权重 1.0
   - `"高山" in ["雪山", "云海", "高山"]` → 命中

## 三、素材-脚本匹配分析

### 方案 A 匹配结果

以目录 `高山/` 下的文件 `雪山-云海.mp4（标签 ["雪山", "云海", "高山"]）` 为例：

```
_score() 计算公式：命中权重之和 / 关键词个数

第1段 keywords=["高山", "山脉"]
  雪山-云海.mp4  → "高山"命中(2.0) + "山脉"未命中 → 2.0/2 = 1.0
  华山-云海.mp4  → "高山"命中(2.0) + "山脉"未命中 → 2.0/2 = 1.0
  泰山-日出.mp4  → "高山"命中(2.0) + "山脉"未命中 → 2.0/2 = 1.0
  → 全部得分相同，任选一个

第2段 keywords=["大海", "海洋"]
  海浪-拍岸.mp4  → "大海"命中(2.0) + "海洋"未命中 → 2.0/2 = 1.0
  海上-日落.mp4  → "大海"命中(2.0) + "海洋"未命中 → 2.0/2 = 1.0
  珊瑚-礁石.mp4  → "大海"命中(2.0) + "海洋"未命中 → 2.0/2 = 1.0

第3段 keywords=["草原", "牧场"]
  辽阔-草原.mp4  → "草原"命中(2.0) + "牧场"未命中 → 2.0/2 = 1.0
  羊群-吃草.mp4  → "草原"命中(2.0) + "牧场"未命中 → 2.0/2 = 1.0
  野马-奔腾.mp4  → "草原"命中(2.0) + "牧场"未命中 → 2.0/2 = 1.0
```

**结论**：三个目录的关键词都在目录名上命中，得分完全相同。选中的素材取决于数据库查询顺序。

### 方案 B 匹配结果（配置了同义词）

```
第1段 keywords=["高山", "山脉"]
  同义词扩展：高山→[mountain, peak, alpine, snow], 山脉→[mountain, range, ridge]
  snow_peak.mp4     → "高山"通过同义词"mountain"命中(1.0) → 1.0/2 = 0.5
  cloud_sea.mp4     → "高山"通过同义词"mountain"命中(1.0) → 1.0/2 = 0.5
  sunrise.mp4       → "高山"通过同义词"mountain"命中(1.0) → 1.0/2 = 0.5

第2段 keywords=["大海", "海洋"]
  同义词扩展：大海→[ocean, sea, wave, marine], 海洋→[ocean, sea, maritime]
  wave_crash.mp4    → "大海"通过同义词"wave"命中(1.0) → 1.0/2 = 0.5
  sunset_sea.mp4    → "大海"通过同义词"sea"命中(1.0) → 1.0/2 = 0.5
  coral_reef.mp4    → "大海"通过同义词"ocean"命中(1.0) → 1.0/2 = 0.5

第3段 keywords=["草原", "牧场"]
  同义词扩展：草原→[grassland, steppe, prairie, meadow], 牧场→[pasture, ranch, grassland]
  vast_steppe.mp4   → "草原"通过同义词"steppe"命中(1.0) → 1.0/2 = 0.5
  sheep_grazing.mp4 → "牧场"通过同义词"grassland"命中(1.0) → 1.0/2 = 0.5
  wild_horse.mp4    → "草原"通过同义词"grassland"命中(1.0) → 1.0/2 = 0.5
```

**注意**：同义词匹配权重只有 1.0，且 `/2` 后得分 0.5。如果想提高得分，可以在 `keywords` 中加入英文字词：
```json
"keywords": ["高山", "山脉", "mountain"]
```
这样脚本关键词 `"mountain"` 精确匹配标签 `"mountain"`，权重 2.0。

## 五、BGM 匹配说明

脚本中 `"style": "knowledge"`，BGM 匹配使用 `Matcher.match_bgm()`，会用中文关键词搜索：

```python
"knowledge": ["知识", "学习", "科普", "教育", "轻快"]
```

所以 BGM 文件名需要包含这些词才能命中。推荐命名：

```
assets/bgm/
├── 科普-轻快.mp3      ← 标签 ["科普", "轻快"] → 匹配 knowledge 风格
├── bgm_知识.mp3        ← 标签 ["bgm", "知识"]  → 匹配
└── 安静-放松.mp3       ← 标签 ["安静", "放松"] → 不匹配（不会被选中）
```

或者在脚本中直接指定 BGM 文件名：
```json
"bgm": "科普-轻快.mp3"
```

## 七、常见调整

| 需求 | 修改方法 |
|------|---------|
| 某段想用特定的视频 | 在该段 `keywords` 中加入该视频文件名中的某个标签词 |
| 不想用目录名匹配 | 用英文目录（如 `mountain/`）避免被追加为标签 |
| 提高同义词匹配分数 | 在 `keywords` 里也加入对应的英文字词 |
| 使用图片代替视频 | 图片同样入库，匹配逻辑相同；渲染器优先用视频 |
| 直接指定 BGM | 脚本顶层 `"bgm": "文件名.mp3"`，不走匹配 |
| 查看已入库的标签 | 打开 GUI → 素材管理 → 查看"标签"列 |
